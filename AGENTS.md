# Project instructions

开始任何任务前，先阅读 `PROJECT_CONTEXT.md`。

处理前端任务时，还必须阅读：

- `frontend/FRONTEND_REQUIREMENTS.md`
- `frontend/FRONTEND_CONSTRAINTS.md`

文档职责：

- 项目背景、阶段、整体边界和技术栈，以 `PROJECT_CONTEXT.md` 为准。
- 前端页面、路由、交互和验收要求，以
  `frontend/FRONTEND_REQUIREMENTS.md` 为准。
- 前端实现方式、代码规范和 Git 约束，以
  `frontend/FRONTEND_CONSTRAINTS.md` 为准。

如果文档之间存在直接冲突，不要自行猜测，先指出冲突。
