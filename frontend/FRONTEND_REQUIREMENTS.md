# 前端需求

> 当前前端接入真实 Go API。整体范围见 [`../PROJECT_CONTEXT.md`](../PROJECT_CONTEXT.md)，接口见 [`../API_CONTRACT.md`](../API_CONTRACT.md)。

## 1. 主流程

```text
登录或注册
  → 完善档案和营养目标
  → 查询或创建 Food
  → 添加饮食记录和可选图片
  → 查看每日汇总和最近 7 天趋势
```

## 2. 路由

| 路由 | 页面 |
| --- | --- |
| `/login` | 登录 |
| `/register` | 注册 |
| `/app/dashboard` | Dashboard |
| `/app/meals` | 饮食记录 |
| `/app/meals/new` | 新建记录 |
| `/app/meals/:id` | 记录详情 |
| `/app/meals/:id/edit` | 编辑记录 |
| `/app/foods` | 食物搜索 |
| `/app/foods/new` | 自定义食物 |
| `/app/foods/:id` | 食物详情 |
| `/app/reports` | 每日汇总和 7 天趋势 |
| `/app/profile` | 档案、目标和偏好 |

除登录和注册外都需要登录。当前没有菜谱库和独立推荐页。

## 3. 页面要求

### 登录和注册

- 登录支持用户名或邮箱、密码。
- 注册包含用户名、邮箱、密码和确认密码。
- 展示字段错误、账号冲突和登录失败。
- 登录成功进入 Dashboard；会话失效时返回登录页。

### Dashboard

- 展示今日热量、蛋白质、脂肪和碳水化合物。
- 展示目标对比、最近记录和简单规则提示。
- 提供“添加饮食记录”入口。
- 无记录或无目标时显示清晰引导。

### 饮食记录

- 填写用餐时间、餐次和一个或多个 Food。
- 每项填写重量或份量。
- 可上传一张 JPEG、PNG 或 WebP 图片，最大 10 MB。
- 前端预览只用于交互，最终营养值以后端计算为准。
- 列表支持日期、餐次和关键词筛选。
- 支持查看、编辑和删除；删除需要确认。

### Food

- 搜索公共 Food 和当前用户的自定义 Food。
- 无结果时可以创建自定义 Food。
- 展示名称、分类、标准份量和每 100g 营养。
- 缺失营养显示“暂无数据”，不能显示成真实零值。
- 当前不展示菜谱原料和做法。

### 营养报告

- 展示选中日期的每日营养汇总。
- 展示以该日期结束的连续 7 天趋势。
- 没有记录的日期仍保留在趋势中。
- 规则提示使用后端返回结果，前端不复制营养判断规则。
- 当前不做 30 天或自定义区间分析。

### 个人设置

- 编辑个人档案、营养目标和饮食偏好。
- 没有目标时仍可查看实际摄入。
- 饮食偏好只保存和展示，不宣传为完整个性化推荐。

## 4. 响应式与交互

- 支持 375px 手机、768px 平板和 1440px 桌面。
- 电脑端使用侧栏，手机端使用顶部栏和底部导航。
- 表格在手机端改为卡片或其他可读布局。
- 页面覆盖加载、空数据、错误、提交中和成功状态。
- 表单有标签和错误提示，不能只用颜色表达营养状态。

## 5. API 边界

- 页面只调用 `lib/api`，基础地址来自 `VITE_API_BASE_URL`。
- 请求、响应、错误、分页、时间和上传遵守 `API_CONTRACT.md`。
- 正常运行使用真实 Go API；Mock 只可用于局部开发辅助，不能替代真实主流程。
- 前端不直接访问 MongoDB、图片存储或服务端密钥。

## 6. 手工验收

至少手工走通：

1. 注册和登录。
2. 编辑档案、目标和偏好。
3. 查询 Food；无结果时创建自定义 Food。
4. 创建、编辑和删除饮食记录。
5. 选择并上传图片。
6. 查看每日汇总和最近 7 天趋势。
7. 检查手机端和电脑端主要页面。

完成前还需通过 TypeScript 检查、lint 和生产构建。
